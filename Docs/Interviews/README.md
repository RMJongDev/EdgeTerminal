# Interviews

Ruwe interview-output per sessie met de klant of stakeholders. Eén markdown-bestand per interview. Gebruik deze bestanden als bron-van-waarheid voor briefings, voorstel-specs en functional design — ze worden nooit samengevat-en-weggegooid.

## Naming

```
YYYY-MM-DD-<onderwerp>-<stakeholder>.md
```

Voorbeelden:
- `2026-04-15-kickoff-klant.md`
- `2026-04-22-processen-operations.md`
- `2026-05-03-tech-constraints-it-lead.md`

## Format per interview

```markdown
# Interview — <onderwerp>

- **Datum:** YYYY-MM-DD
- **Deelnemers:** <namen + rollen>
- **Doel:** 1 zin — wat wil je uit dit gesprek halen?
- **Gerelateerde fase:** 1 | 2 | 3 | 4

## Vragen & antwoorden

### <categorie 1>
- **V:** ...
- **A:** ...

### <categorie 2>
- **V:** ...
- **A:** ...

## Aannames & open punten
- > AANNAME: ...
- ❓ OPEN: ...

## Actiepunten
- [ ] ...
```

Markeer kritieke vragen met 🔴 en optionele met 🟡 zoals elders in de accelerator.
