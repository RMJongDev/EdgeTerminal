# Backlog

> Levend backlog-document met epics en user stories. Wordt tijdens fase 4 (build-prep) voor het eerst gevuld en daarna per sessie bijgewerkt. Nieuwste epics bovenaan; binnen een epic staan stories in prioriteitsvolgorde.

## Format

```
## EPIC-<nr> — <korte titel>
**Doel:** 1 zin waarom deze epic bestaat.
**Status:** todo | in-progress | done
**Definition of done:** meetbare afrondingscriteria.

### STORY-<nr> — <korte titel>
- **Als** <rol>, **wil ik** <actie>, **zodat** <waarde>.
- **Acceptatiecriteria:**
  - [ ] criterium 1
  - [ ] criterium 2
- **Notities:** technische haken, afhankelijkheden, open vragen.
- **Status:** todo | in-progress | done
```

## Labels

- `🔴 kritiek` — blokkeert release
- `🟡 belangrijk` — nodig voor MVP maar niet blocking
- `🟢 nice-to-have` — post-MVP
